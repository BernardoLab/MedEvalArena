SCOPE / CONSTRAINTS
- Treat this as a “single best answer” item.
- Judge ONLY what is written.
- Do NOT rewrite the question.

- Use only information in the vignette + widely accepted medical knowledge. Do NOT invent missing facts.

TASK
Produce three outputs:
A) Medical and logical analysis (string)
B) medical_accuracy_score (integer: 1–5)
C) logical_validity (boolean: true/false)

A) Medical + logical audit (board-exam style)
You are auditing a medical multiple-choice question for (1) best-answer defensibility, (2) logical validity, and (3) medical accuracy.
- If the stem is insufficient, internally inconsistent, or allows >1 reasonable best answer, explicitly flag it.
- Explain reasoning concisely (exam-explanation style): cite the specific stem clues that drive decisions; avoid long hidden chain-of-thought.

LOGICAL VALIDITY FAILURE MODES (label exactly one primary mode; add secondary if needed)
- C = Contradiction: internal inconsistency makes the item incoherent/unsolvable.
- N = No answer defensible: none of the options can be defended as best from the vignette.
- M = Multiple answers defensible: two or more options are equally/near-equally defensible as best.
- U = Underspecified: missing a critical detail; requires an unstated assumption to identify a unique best answer.
- K = Miskeyed: a unique best answer exists, but the provided key is not that answer.
- V = Valid: none of the above; item supports a unique best answer and the key matches it.

SUBTASKS
1) Best-answer defensibility (stepwise)
   a) Restate the clinical problem in 1 sentence (what the question is truly asking).
   b) Extract the key discriminating findings from the stem (positive + negative clues).
   c) Identify the most likely diagnosis/mechanism/next step implied by those findings.
   d) Evaluate the KEYED_ANSWER: why it fits; note any assumptions required.
   e) Rapidly assess each alternative choice: 1–2 stem-based reasons it’s worse/incorrect.

2) Logical validity classification (use the failure modes above)
   - Determine whether the stem + choices allow selection of ONE best option under standard exam framing.
   - Assign a primary label: {V, C, N, M, U, or K}.
   - Justify the label with specific references to missing/contradictory details, option overlap, or key mismatch.
   - If applicable, list secondary label(s) that also apply (e.g., “U leading to M”).

3) Medical accuracy checks
   - Is the keyed answer consistent with current mainstream standards (pathophys, diagnosis, management)?
   - Does the item’s implied logic contain inaccuracies (wrong associations, outdated guidance, incorrect contraindications)?
   - If correctness depends on guideline nuance or regional variation, state that and how it affects “best answer.”

4) Final output (for JSON `analysis` field)
Produce a concise audit summary in <=150–220 words with this structure:
- Validity label: {V/C/N/M/U/K} (and any secondary label(s) if relevant)
- Verdict on keyed answer: {Correct best / Miskeyed / Not uniquely determinable / No defensible answer}
- Key stem clues that drive (or fail to drive) a unique best answer
- Main logical issue(s), if any (tied to the label definitions)
- Main medical accuracy issue(s), if any
- Minimal fix: 1 concrete edit to the stem/choices/key that would make it unambiguous and accurate


B) DIMENSION 1 — medical_accuracy_score (integer 1–5)
Score how well the vignette/options/intended teaching point align with established medical consensus and typical board-style guidance (standard of care, major guidelines, medical board-level expectations).

1 = Dangerous/False: clear hallucinations or advice that plausibly causes harm; strongly contradicts standard care.
2 = Inaccurate: major medical errors or clearly outdated guidance that would mislead a learner.
3 = Minor issues: mostly correct but missing important details, uses nonstandard terminology, or is clinically unrealistic in a non-critical way.
4 = Accurate: consistent with current standard care/guidelines; clinically sensible.
5 = Gold standard: fully accurate, precise, realistic, and board-relevant.

Safety note: “Unsafe” is not the same as “inaccurate.” Only treat as dangerous if it could plausibly lead to harmful real-world action.

C) DIMENSION 2 — logical_validity (boolean)
Set logical_validity = true ONLY if ALL conditions are met:
1) Single best answer: exactly one option is defensible as best (no ties/near-ties).
2) Key supported: the provided key equals that single best answer.
3) No contradictions: nothing in the vignette/options makes the item internally inconsistent or unsolvable.
4) Options permit discrimination: options are comparable enough that a single-best choice is possible.

D) DIMENSION 3 — logical_false_reason (string code)
If logical_validity = true → logical_false_reason MUST be "T".
If logical_validity = false → choose ONE PRIMARY code using this selection rule:

Miskey special-case:
- If (1), (3), and (4) hold BUT (2) fails → "K".

Otherwise choose the most decisive cause using this priority order:
"C" > "N" > "M" > "U"

Codes:
- C = Contradiction: internal inconsistency makes the item incoherent/unsolvable.
- N = No answer defensible: none of the options can be defended as best from the vignette.
- M = Multiple answers defensible: two or more options are equally/near-equally defensible as best.
- U = Underspecified: missing a critical detail; requires an unstated assumption to identify a unique best answer.
- K = Miskeyed: a unique best answer exists, but the provided key is not that answer.
- T = Used only when logical_validity is true.

Respond with exactly one JSON object, no markdown, no extra text:
